大O记号的刻度

常数复杂度 O( 1 )

算法不含随输入规模变换的转向（循环、调用、递归等），且顺序执行，则说这个算法的复杂度为 O(1)。

对数复杂度 O( log^c(n) )

在此种情况中，对数的底数常常会被忽略，因为往往可以通过数学变换来改变对数的底数。

这类算法非常高效，复杂度无限接近于常数。

多项式复杂度 O( n^c )

此种复杂度由于影响因素往往取决于最高次幂项，因此较低次幂的项往往忽略不计。

其中，当幂次 c == 1 时，表示算法的时间复杂度与 n 线性相关，因此称 O(n) 为线性复杂度。

指数复杂度 O( 2^n )

当一个算法的计算成本按指数增长时，往往我们认为此种算法为问题的难解。

冒泡排序

课程通过冒泡排序来描述一个算法的可行性以及算法的好坏对计算效率的影响，此处将简要描述冒泡排序。

/**
 * 冒泡排序
 */
class BubbleSort {

    // 课程演示中冒泡排序的写法，因增加了反转判断，平均效率更高
    public static void bubbleSort1(int[] nums) {
        int n = nums.length;
        for (boolean sorted = false; sorted = !sorted; n--) { // 每次遍历数组后对sorted进行反转来查看是否排序完成
            for (int i = 1; i < n; i++) {
                if (nums[i] < nums[i - 1]) {
                    int temp = nums[i];
                    nums[i] = nums[i - 1];
                    nums[i - 1] = temp;
                    sorted = false; // 如果数组中仍有反转，说明排序未完成
                }
            }
        }
    }

    // 常规冒泡排序写法，效率较低
    public static void bubbleSort2(int[] nums) {
        int n = nums.length;
        for (int i = 0; i < n; i++) {
            for (int j = 1; j < n - i; j++) {
                if (nums[j - 1] > nums[j]) {
                    int temp = nums[j];
                    nums[j] = nums[j - 1];
                    nums[j - 1] = temp;
                }
            }
        }
    }
    
}

迭代与递归

本部分通过简要介绍减治法、分治法和动态规划，来体现迭代和递归。

注意：此部分不会详细讲述上述三种算法。

减治法

通过将一个大规模的问题分解为两个子问题，将其中一个子问题解决，此后递归减治另一子问题，由子问题的解得到原问题的解。

/**
 * 求和、数组翻转
 */
class Test {
    
    // 减治法递归求和
    public static int sum(int[] nums, int n) {
        return n > 1 ? sum(nums, n - 1) + nums[n - 1] : nums[n - 1];
    }
    
    // 减治法递归翻转数组
    public void reverse(int[] nums, int begin, int end) {
        if (begin < end) {
            int temp = nums[begin];
            nums[begin] = nums[end];
            nums[end] = temp;
            reverse(nums, begin + 1, end - 1);
        }
    }
    
}

分治法

通过将一个大规模的问题分解为两个子问题，并不断分解其子问题，直到最新的子问题得到解决后，依次得到原问题的解。

动态规划

自底向上，通过先求出子问题来推出原问题的解。

/**
 * 斐波那契数列
 */
class FB {
    
    // 分治 O(2^n)
    public static int fib(int n) {
        return n > 1 ? fib(n - 1) + fib(n - 2) : n;
    }
    
    // 动态规划 O(n)
    public static int fib(int n) {
        if (n == 0) return 0;
        int f = 0; // fib(0)
        int g = 1; // fib(1)
        while (n-- > 1) {
            g = g + f;
            f = g - f;
        }
        return g;
    }

}

动态规划典型例题：最长公共子序列

LeetCode：最长公共子序列

此处介绍了两种求得最长公共子序列的方法：分治法和动态规划。

/**
 * 最长公共子序列
 */
class LCS {

    // 分治法递归 O(2^n)
    public static int lcs1(String s1, String s2, int l1, int l2) {
        if (l1 == 0 || l2 == 0) return 0; // 递归基
        if (s1.charAt(l1 - 1) == s2.charAt(l2 - 1)) { // 如果判断此处字符串相同，同时除去两个字符，结果 + 1
            return lcs1(s1, s2, l1 - 1, l2 - 1) + 1;
        }
        // 如果此处字符串不相等，分两个方向寻找子序列
        int result1 = lcs1(s1, s2, l1 - 1, l2);
        int result2 = lcs1(s1, s2, l1, l2 - 1);

        // 返回两个结果中比较大的值
        return Math.max(result1, result2);
    }

    // 动态规划 O(s1.length * s2.length)
    public static int lcs2(String s1, String s2) {
        // 创建一个dp表，用于记录动态规划过程中产生的中间结果，length + 1 确保每个数据位都可以进行取值操作
        int[][] dp = new int[s1.length() + 1][s2.length() + 1];
        for (int i = 1; i < dp.length; i++) {
            char c1 = s1.charAt(i - 1);
            for (int j = 1; j < dp[i].length; j++) {
                char c2 = s2.charAt(j - 1);
                if (c1 == c2) {
                    // 如果相等，取左上角的值并且 + 1，因为左上角的值一定是最长公共子序列的组成部分。
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    // 如果不相等，取左和上中较大的值作为结果
                    dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }
        // 返回最终结果
        return dp[s1.length()][s2.length()];
    }

}
